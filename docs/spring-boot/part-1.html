STEP 1 — INTRODUCTION

What is
Spring Boot?

Before you write a single line of code, you need to understand the problem Spring Boot solves — and why that problem was so painful that it changed the whole Java ecosystem.

The problem it solves

Spring without Boot was painful

Spring Framework has been the gold standard for Java enterprise apps since 2003. But using it meant writing hundreds of lines of configuration before a single endpoint could run.

Toggle between the two approaches below to feel the difference yourself.

// web.xml — deploy descriptor (required)
<web-app> <servlet> <servlet-name>dispatcher</servlet-name> <servlet-class>DispatcherServlet</servlet-class> <init-param> <param-name>contextConfigLocation</param-name> <param-value>/WEB-INF/spring-mvc.xml</param-value> </init-param> <load-on-startup>1</load-on-startup> </servlet> <servlet-mapping> <servlet-name>dispatcher</servlet-name> <url-pattern>/</url-pattern> </servlet-mapping> </web-app>
// spring-mvc.xml — more config
<beans> <context:component-scan base-package="com.shop" /> <mvc:annotation-driven /> <bean class="InternalResourceViewResolver"> <property name="prefix" value="/WEB-INF/views/" /> <property name="suffix" value=".jsp" /> </bean> </beans>

And this is before you've written a single controller. You still need to configure a DataSource, TransactionManager, EntityManagerFactory…

// The entire app — one file, zero XML
@SpringBootApplication public class MyShopApplication { public static void main(String[] args) { SpringApplication.run(MyShopApplication.class, args); } } // That's it. Your app is running on port 8080.
// Add an endpoint
@RestController public class HelloController { @GetMapping("/hello") public String hello() { return "Hello, World!"; } }

No XML. No WAR files. No application server. Hit localhost:8080/hello and you're done.

⚡
The old approach wasn't wrong — it was just tedious. Every project copied the same boilerplate. Spring Boot packages all of that into sensible defaults you can override when needed.

Side by side

Spring Framework vs Spring Boot

Spring Boot doesn't replace Spring — it sits on top of it. Here's the honest comparison.

What Spring Framework Spring Boot
Configuration Manual — XML or Java @Bean methods Auto-configured from classpath
Server setup Deploy WAR to external Tomcat/Jetty Embedded Tomcat in the JAR — just run it
Dependencies Pick and version every library yourself Starters bundle compatible versions for you
Time to first endpoint 30–60 minutes of setup Under 5 minutes
Flexibility Full control over everything Same control — defaults are just overridable
💡
Spring Boot doesn't hide anything. Everything it auto-configures, you can override. As you learn more, you'll peel back the layers — but you don't have to on day one.
