The generated code

One class. One annotation. That's the whole app.

Unzip the download, open it in IntelliJ, and look at the only Java file Initializr generated. It's smaller than you expect.

// src/main/java/com/shop/MyShopApplication.java @SpringBootApplication public class MyShopApplication { public static void main(String[] args) { SpringApplication.run(MyShopApplication.class, args); } }

That's the entire app. @SpringBootApplication does all the heavy lifting. Click each part below to understand what it actually means.

@SpringBootApplication
is shorthand for three annotations combined:
@SpringBoot­Configuration
Marks this class as a source of bean definitions
@EnableAuto­Configuration
Turns on Spring Boot's auto-configuration engine
@Component­Scan
Finds all your classes annotated with Spring annotations
← Click a card to see what each annotation does.
⚡
You almost never need to split these three apart. Use @SpringBootApplication and forget the internals — unless you need to disable auto-configuration for a specific class (rare, advanced).

Verify it's running

Make your first HTTP request

Your app is live on port 8080. Try hitting it — but read the response carefully. The 404 you'll get is actually good news.

Open your browser and navigate to:

GET http://localhost:8080
404 Not Found
{
  "timestamp": "2024-01-15T10:23:52.000+00:00",
  "status": 404,
  "error": "Not Found",
  "path": "/"
}
$ curl -i http://localhost:8080
HTTP/1.1 404 Not Found
Content-Type: application/json

{"timestamp":"2024-01-15T10:23:52.000+00:00",
 "status":404,"error":"Not Found",
 "path":"/"}
$ http GET localhost:8080
HTTP/1.1 404 Not Found
{
  "error": "Not Found",
  "path": "/",
  "status": 404,
  "timestamp": "2024-01-15T10:23:52.000+00:00"
}
✅
A 404 from Spring Boot is success. It means the server is running and received your request — it just has no endpoint mapped to / yet. Spring Boot even formats the error as JSON automatically. A "connection refused" would mean the server didn't start. A 404 means it did.
🚫
Port 8080 already in use? Something else is running on that port. Either stop it, or add server.port=8081 to src/main/resources/application.properties and restart.
